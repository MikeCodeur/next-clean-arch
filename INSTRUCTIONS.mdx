## Comprendre

Pour le moment l’authentification est repartie dans de nombreux fichiers

- `auth.ts` (de next-auth)
- `crypt.ts`

Ainsi que plusieurs fonctions comme

- `getSession`, `isAuth`, `isAuthAdmin` etc …
- `sign-up`

S’il l’on souhaite changer facilement de système d’authentification, tout doit être regrouper dans un module / service, isolé.

Nous allons créer cela dans le répertoire

- `services/authentification`

## Exercice

Pour simplifier l’exercice nous avons regroupé toutes les fonctions liées de prêt ou de loin à l’authentification dans `exercise/auth-util.ts` , toujours dans la présentation.

- **🐶 Dans cet exercice tu vas devoir tout déplacer dans le service authentification**

Fait un `pnpm build` pour voir toutes les `errors` d’importation et modifie les `imports`

Fichiers

- `service/authentification/auth-service`
- `exercise/auth-util.ts`
